Tomcat & Servlet

Tomcat 是一个开源的 Servlet 容器（Web 服务器），用于运行 Java Web 应用程序。

核心作用：

  • 接收 HTTP 请求，解析并交给 Servlet 处理。
  • 管理 Servlet 生命周期（初始化、调用、销毁）。
  • 处理静态资源（HTML、CSS、JS）和动态资源（JSP、Servlet）。

Servlet 是 Java 编写的服务端程序，用于处理客户端（如浏览器）的 HTTP 请求并生成动态响应。它是 Java Web 开发的核心基础。

核心特性：

  • 运行在 Servlet 容器（如 Tomcat）中。
  • 通过 javax.servlet 和 javax.servlet.http 包提供 API。
  • 支持多线程处理请求（每个请求由独立线程处理）。

Servlet

  • Servlet就是sun公司开发动态web的一门技术。

  • Sun在这些API中提供一个接口叫做：Servlet，如果你想开发一个Servlet程序，只需要完成两个小步骤：

    1. 编写一个类，实现Servlet接口；
    2. 把开发好的Java类部署到web服务器中。

    把实现了Servlet接口Java程序叫做，Servlet

    1. Servlet 的核心任务是处理请求逻辑，而非简单对应页面。

    2. 通过合理设计，一个 Servlet 可灵活驱动多个页面，或完全不涉及页面（如纯 API）。

    3. 现代框架（如 Spring MVC）进一步抽象了 Servlet，开发者通过 @Controller 和 @RequestMapping 更直观地管理请求与视图的关系。

Serlvet接口Sun公司有两个默认的实现类：HttpServlet，GenericServlet

Servlet原理

ServletContext

ServletContext 是 Java Web 应用中所有 Servlet 共享的上下文对象，代表整个 Web 应用的运行环境。所有用户和所有 Servlet 共享同一个 ServletContext 实例。

核心功能与使用场景

（1）共享全局数据

  • 存储全站配置（如数据库连接池、文件上传路径）。
  • 统计网站访问次数。

（2）读取 Web 应用资源

  • 加载配置文件（如数据库参数）。
  • 读取静态资源（如 XML 模板）。

（3）获取应用初始化参数

（4）监听应用事件

（5）请求转发

Cookie & Session

无状态的会话：用户打开一个浏览器，点击了很多超链接，访问多个web资源，关闭浏览器，这个过程可以称之为会话。
有状态的会话：一个用户打开一个浏览器，访问某些资源（网站），下次再来访问该资源（网站），我们会知道这个用户曾经来过，称之为有状态会话；

  1. 服务端给客户端一个信件，客户端下次访问服务端带上信件就可以了；cookie（客户端）
  2. 服务器登记你来过了，下次你来的时候我来匹配你；seesion（服务端）

cookie：

  • 客户端技术，（响应、请求）

  • 从客户端请求中拿到cookie

  • 服务器响应给客户端cookie

session（重点）：

  • 服务端技术，利用这个技术，可以保存用户的会话信息？我们可以把信息或者数据放在Session中

  • 服务器会给每一个用户（浏览器）创建一个Seesion对象。

  • 一个session独占一个浏览器，只要浏览器没有关闭，这个session就存在。

  • 用户登录之后，整个网站它都可以访问。（保存用户的信息（例如B站进入每个页面无需重新登陆）；也可以保存购物车的信息）

概念 定义 核心作用
Cookie 由服务器生成并发送到用户浏览器的小型文本文件，存储在客户端。 用于在客户端保存用户状态信息（如登录状态、偏好设置）。
Session 服务器端创建的临时存储空间，用于保存用户在一次会话中的交互数据。 在服务器端安全存储敏感或临时用户数据（如购物车内容）。
  • Cookie 是客户端的“轻量级便签”，适合存储非敏感信息。
  • Session 是服务器端的“保险箱”，适合保护关键数据。
  • 两者常结合使用：Session ID 通过 Cookie 传递，实现安全的状态管理。
  • 根据需求选择技术：小型偏好用 Cookie，敏感操作依赖 Session，分布式系统考虑 Token。

Maven

Maven 项目默认会生成 target 目录，它是 Maven 构建过程中所有生成文件（编译结果、测试报告、打包文件等）的默认输出目录。

  • target 是 Maven 的标准构建输出目录，存放编译结果、测试报告和打包文件。
  • 无需手动管理，Maven 会自动维护其内容。
  • 不要提交到版本控制，因为它可以通过重新构建生成。
